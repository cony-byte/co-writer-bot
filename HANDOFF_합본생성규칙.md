# 핸드오프 — 합본(에피소드 컴파일) 생성 규칙

co-writer-bot이 그 회차 컷 영상들을 하나로 이어붙여 '합본' mp4를 만드는 현재 규칙 정리
(2026-07-22). 핵심 코드: `bot/dispatch_storyboard.py::_do_compile/_run_compile`,
`bot/edit_plan.py`(편집 계획), `bot/episode_compile.py`(ffmpeg 렌더).

핵심 요약: **편집 전략은 LLM에 안 맡기고(러프), 있는 컷을 씬→컷 순서대로 이어붙인다.
말·행위 안 끊김이 최우선, 콘티 의도 길이는 후순위. 렌더는 ffmpeg CLI.**

---

## 1. 트리거 / 대상
- `[합본] <작품> N화` 또는 "합본해줘". **화 번호 필수**(메시지 → conti_state 순).
- 콘티는 노션/저장분에서 자동 로드. 씬 필터(`씬2,3` / `씬2-4`)로 일부 씬만 가능.
- 영상화 안 된 씬이 있으면 "그래도 있는 컷만으로?" 확인 버튼 후 진행.

## 2. 편집 계획 = 러프 플랜 (LLM 안 씀)
`edit_plan.build_edit_plan` → 실제로는 `_build_rough_plan`.
- **있는 컷을 씬 순서·컷 번호 순서 그대로** 한 비디오 트랙에 이어붙임.
- 각 컷은 **실측 길이 전체 + 하드컷**이 기본.
- ffprobe로 실측, 길이 0인 컷은 드롭.
- (LLM 편집전략 코드 `_SYSTEM/_validate_plan` 등은 남아있지만 호출 안 함 — "플랜이 자꾸
  이상하게 자른다"는 사용자 요청으로 우회.)

## 3. 트림 우선순위 (★말·행위 안 끊김 최우선)
- **음성 기반 꼬리 트림**: 대사 컷은 seedance가 실제 발화 음성을 넣으므로(`generate_audio`),
  ffmpeg `silencedetect`로 **마지막 발화 끝**을 검출해 그 뒤 **정적 꼬리만** 잘라낸다
  (`edit_plan._last_speech_end`). 여운 +0.4초 남김.
- **보호선**: 트림해도 (a) 발화 끝+여운, (b) 콘티 의도 길이(`[N초]`) 둘 다를 최소 보장 →
  발화·행위 길이를 지킴.
- **콘티 의도 길이로 '줄이는(cap)' 로직은 제거됨** — 실측이 의도보다 길어도 여분에 행위가
  이어질 수 있어 함부로 안 자름(우선순위: ①말 ②행위 ③의도 길이).
- **오디오 없는 무음 컷**: silencedetect 검출 불가 → **아무것도 안 자르고 실측 전체 유지.**

## 4. 렌더 (ffmpeg CLI, `episode_compile._render`)
- 각 컷을 `libx264 veryfast`로 해상도·fps 통일(`COMPILE_WIDTH=1080`, `HEIGHT=1920`, `FPS=30`) →
  `concat` demuxer로 이어붙임.
- **트랜지션**: `transition_in=="fade"`인 컷만 fade-in/out을 미리 구워 넣음(대부분 하드컷).
- **배속(speed)**: 컷별 `speed`(0.5~2.0) → `setpts`(영상)+`atempo`(오디오). 출력 길이=소스/속도로
  cursor·total 재계산. fade는 배속 뒤(출력 타임라인) 기준.
- **오디오**: 컷 자체 오디오(대사/앰비언트)가 있으면 타임라인 위치에 맞춰 추출·믹스. 없으면 무음.
  나레이션 TTS는 현재 비활성(타이밍 이슈). 오디오가 하나도 없으면 무음 mp4.
- **배경음악**: `OPENROUTER_MUSIC_ENABLED`(기본 ON) — 켜져도 **합본 mp4에 안 섞고 별도 mp3**로만
  올림(사용자가 나중에 직접 입힐지 결정).

## 5. 확정
- draft로 업로드 → 카드 버튼: `[✅ 최종본으로 확정]` / `[✂️ 편집 요청]` / `[🔄 다시 만들기]`.
- 확정 시 `draft_*.mp4` → `_최종.mp4` 승격, 같은 화의 다른 draft/최종본 삭제(최종본 1개 유지).

## 6. 편집 요청 (합본 카드에서, 누적)
`[✂️ 편집 요청]` → 자유 답변 → 제약 LLM(`edit_plan.apply_edit_feedback`, Sonnet 5)이 현재 plan에
**사용자가 명시한 편집만** 적용 → 재렌더. 지원:
- 길이 조정 / 컷 제거 / 순서 변경 / 배속(0.5~2.0) / 트랜지션(cut/fade)
- 안전장치: 원본 컷만 참조, 새 컷 창작·시키지 않은 재단 금지, duration은 소스 최대로 클램프.
- 편집 답변은 양 백엔드 공통 결정적 게이트로 라우터보다 먼저 잡음. 편집은 누적(base_plan).

## 7. 지금 안 하는 것 / 함정
- **똑똑한 편집(LLM 자동 컷 선택)은 꺼져 있음** — 대사·행위를 마음대로 재단하던 문제로 우회.
- **나레이션 TTS 믹싱** 비활성, **배경음악은 합본에 직접 안 섞음**(별도 mp3).
- 러프 플랜이라 **컷 순서=씬/컷 번호 순서 그대로** — 컷이 잘못된 순서로 저장돼 있으면 합본도 그대로.
- silencedetect는 오디오 있는 컷에만 작동. 무음 컷은 트림 안 함(의도).

> 관련: `HANDOFF_영상화_인물일관성.md`(컷 영상 생성), CapCut 내보내기/편집은 `[페이컷]`/`[페이컷편집]`.
