# 핸드오프 — "작품명 끌어오기"(스레드/노션 링크 → 작품 자동 해석) 이식 가이드

> co-writer-bot의 작품 해석 로직을 **다른 슬랙 봇**에 옮기기 위한 문서.
> 목표: 사용자가 `<작품>`을 매번 안 써도, **① 별칭 ② 스레드 맥락 ③ 노션 링크**로 작품을 알아서 찾게 한다.

---

## 1. 전체 그림 (해석 우선순위)

명령이 들어오면 작품을 이 순서로 결정한다:

1. **명령에 `<작품>` 명시** → `works.resolve(name)` (별칭이면 정식명으로 변환)
2. **명령에 노션 링크** → 등록/동기화 후 그 작품 (선택: `[동기화]`/pre-hook)
3. **`<작품>` 없음** → `_work_from_thread(스레드 전체 텍스트)`
   - 스레드에 노션 링크 있으면 → `page_id`로 등록된 작품 찾기
   - 없으면 → 스레드의 첫 `<작품>` 토큰(단, URL은 제외)

핵심 부품 2개: **`works.py`(등록소)** + **`_work_from_thread()`(스레드 회수)**.

---

## 2. 그대로 복사할 파일: `bot/works.py`

작품·별칭 → 노션 page_id 매핑 저장소. `data/notion_pages.json`에 영구 저장(재시작 무관).
의존성: `config.BASE_DIR`, `config.NOTION_PAGES`(env JSON, 없으면 `{}`)뿐 — 그대로 이식 가능.

```python
# 저장 형식: { "<정식작품명>": {"page": "<32자리 page_id>", "aliases": [...]} }
def all_works() -> dict          # 파일 + env NOTION_PAGES 병합(파일 우선)
def resolve(name) -> str|None    # 이름/별칭 → 정식 작품명
def work_by_page(page_id) -> str|None   # page_id(대시 무시) → 정식 작품명  ← 링크 해석의 핵심
def page_of(name) -> str|None    # 작품 → page_id
def register(work, page_id, aliases=None)   # 등록/갱신
def add_aliases(work, aliases) -> str|None  # 별칭 추가
def sanitize(name) -> str        # 시트 탭명 못 쓰는 문자([]:*?/\) 제거 — 노션 제목→작품명용
```

> **주의**: `work_by_page`는 page_id에서 대시(`-`)를 제거하고 비교한다. 노션 URL·API가 대시 유무를 섞어 쓰기 때문. `resolve`는 정식명이면 그대로, 별칭이면 매핑을 뒤진다.

## 3. 그대로 복사할 함수: `_work_from_thread()`

```python
def _work_from_thread(joined: str) -> str | None:
    """스레드 텍스트에서 작품 회수: ①노션 링크 → 등록 작품 ②<작품> 토큰(URL 제외)."""
    lm = re.search(r"https?://\S*notion\.\S+", joined or "")
    if lm:
        from bot import notion_sync
        pid = notion_sync.extract_page_id(lm.group(0))
        if pid:
            w = works.work_by_page(pid)
            if w:
                return w
    for w in re.findall(r"<\s*([^>]+?)\s*>", joined or ""):
        w = w.strip()
        if w.startswith("http") or "notion." in w:   # 슬랙이 감싼 <https://…> 오인 방지
            continue
        return w
    return None
```

- `joined` = 스레드 메시지 본문을 `"\n".join(...)` 한 것.
- `extract_page_id`는 URL/ID에서 32자리 hex를 뽑는다:
  ```python
  m = re.search(r"([0-9a-fA-F]{32})", (url_or_id or "").replace("-", ""))
  return m.group(1) if m else None
  ```

## 4. 명령 핸들러에 붙이는 패턴 (복붙 템플릿)

```python
q = rest.strip()
work = None
wm = SUB_RE.match(q)            # SUB_RE = re.compile(r"^\s*<\s*([^>]+?)\s*>\s*(.*)$", re.S)
if wm:
    work = works.resolve(wm.group(1).strip()) or wm.group(1).strip()
    q = wm.group(2).strip()
if not work:                   # <작품> 없으면 스레드(첫 댓글의 작품/노션 링크)에서 회수
    joined = "\n".join(m["content"] for m in _thread_messages(channel, thread_ts))
    w = _work_from_thread(joined)
    if w:
        work = works.resolve(w) or w
# 이후 work로 시트/바이블 로드
```

co-writer에선 `[생성]`·`[피드백]`·`[확인]`·`[변환]`·`[스토리보드]`가 모두 이 패턴을 공유한다.

### (선택) 명령어에 노션 링크가 섞인 경우 자동 등록
바이블 쓰는 명령 앞단에서, 명령문에 노션 링크가 있으면 **먼저 등록·동기화하고 `<작품>`을 주입**한다:
```python
_WORK_CMDS = CMD_GEN | CMD_FEEDBACK | ...   # 링크를 '소스'로 읽는 명령만 (기록 대상인 [기획]은 제외!)
if cmd in _WORK_CMDS and NOTION_TOKEN:
    rest = re.sub(r"<(https?://[^>|]+)(?:\|[^>]*)?>", r"\1", rest)   # 슬랙 <url|label> 언랩
    lm = re.search(r"https?://\S*notion\.\S+", rest)
    if lm:
        w = _autosync_link(...)          # extract_page_id → work_by_page or sanitize(title) → register + sync
        rest = re.sub(r"https?://\S*notion\.\S+", "", rest).strip()
        if w and not SUB_RE.match(rest): # <작품> 없이 링크만 준 경우 → 작품명 주입
            rest = f"<{w}> {rest}".strip()
```

### (선택) 명령어 없이 링크만 던지면 = 동기화
```python
if not cmd_matched and NOTION_TOKEN and re.search(r"https?://\S*notion\.\S+", query):
    _do_sync(channel, thread_ts, query)   # 링크 읽기 성공=등록, 신규면 제목으로 새 작품
    return
```

## 5. 노션 읽기 최소 의존성 (`notion_sync`에서 가져올 것)

- `extract_page_id(url_or_id)` — 위 4번 참고
- `page_title(page_id, token)` — GET `pages/{id}` → title 프로퍼티. **읽기 성공 = 통합(MCP) 연결 확인** 겸함
- `page_text(page_id, token)` — 블록 순회 → 평문 (선택; 내용까지 당길 때)
- 토큰: 노션 **내부 통합(integration) 토큰**(`ntn_...`), 헤더 `Authorization: Bearer`, `Notion-Version: 2022-06-28`
- ⚠️ **그 페이지가 통합에 연결돼 있어야** 읽힌다(아니면 404). 실패 시 사용자에게 "페이지 `•••` → 연결 → 통합 추가" 안내.

## 6. 이식 시 함정 (실제로 겪은 것)

1. **슬랙 URL 래핑**: 슬랙은 URL을 `<https://…>` 또는 `<https://…|라벨>`로 감싼다.
   - `<작품>` 토큰 파서(`<...>`)가 이걸 작품명으로 오인함 → `_work_from_thread`에서 `startswith("http")`/`"notion." in` 로 제외 필수.
   - 명령 앞단에선 `re.sub(r"<(https?://[^>|]+)(?:\|[^>]*)?>", r"\1", rest)`로 언랩.
2. **별칭 resolve는 모든 진입점에**: `<작품>`뿐 아니라 스레드 회수·자동주입 결과도 `works.resolve(w) or w`로 한 번 더 정규화.
3. **page_id 대시**: 항상 대시 제거 후 비교(`work_by_page`가 처리).
4. **기록 대상 링크는 제외**: 링크를 "읽어올 소스"로 쓰는 명령과 "써넣을 대상"으로 쓰는 명령을 구분. 후자([기획] 같은)를 자동등록 pre-hook에 넣으면 안 됨.
5. **`data/notion_pages.json`은 런타임 상태** → gitignore.
