# 핸드오프 — 다른 봇(storyboard-bot 등)이 작품/별칭/노션을 co-writer와 똑같이 읽게 하기

## 증상
storyboard-bot에서 `<코니>` 같은 **별칭**을 못 알아듣고, 그 작품의 **노션 페이지**도 못 읽음.

## 원인 (핵심)
작품↔노션 매핑·별칭은 co-writer-bot의 **`data/notion_pages.json`** 한 파일에만 있다.
storyboard-bot은 **별도 프로세스/별도 repo**라 이 파일도, `works.py`의 resolve 로직도, 노션 토큰도 없다.
→ 그래서 별칭 해석(`코니`→`cony 테스트 작품`)도, `works.page_of()`로 page_id 찾기도 안 됨.

세 조각이 다 있어야 읽힌다: **① 등록소 파일 ② resolve 코드 ③ 노션 토큰(+페이지 연결)**.

---

## 해결 (권장: 등록소 '공유', 코드 '복사')

### 1) 등록소 파일을 공유 — 두 봇이 같은 notion_pages.json을 보게
`data/notion_pages.json` 형식:
```json
{ "cony 테스트 작품": { "page": "398beda6e49080e1ba4fc3ed14473e6c", "aliases": ["코니"] } }
```
- co-writer가 `[동기화]`/`[기획]`/`[별칭]`으로 여기 쓴다. storyboard-bot은 **읽기만** 하면 됨.
- 방법 A (간단): storyboard-bot의 `works.py`가 이 파일 경로를 env로 받게 →
  `WORKS_PATH = os.environ.get("COWRITER_WORKS_PATH", <기본>)`, 그리고 storyboard-bot 실행 시
  `COWRITER_WORKS_PATH=/Users/cony/Desktop/Github/co-writer-bot/data/notion_pages.json` 지정.
- 방법 B: 심볼릭 링크 `ln -s .../co-writer-bot/data/notion_pages.json .../storyboard-bot/data/notion_pages.json`.
- ⚠️ `notion_pages.json`은 gitignore 대상(런타임 상태)이라 repo로는 안 옮겨진다 — 경로 공유가 정답.

### 2) resolve 코드 복사 — co-writer의 `bot/works.py` 그대로
의존성은 `config.BASE_DIR`/`config.NOTION_PAGES`뿐. 그대로 복붙 가능. 쓰는 함수:
```python
works.resolve(name)      # '코니'/'cony 테스트 작품' → 정식 작품명 (별칭 해석)  ← 이게 없어서 별칭 못 읽던 것
works.page_of(name)      # 작품/별칭 → 노션 page_id
works.work_by_page(pid)  # page_id → 정식 작품명
works.all_works()        # 파일 + env NOTION_PAGES 병합
```
> `_PATH`(notion_pages.json 경로)만 위 1)의 공유 경로로 맞춰라.

### 3) 노션 읽기 — `bot/notion_sync.py` 복사 + 토큰
- storyboard-bot `.env`에 **`NOTION_TOKEN=ntn_...`** (co-writer와 같은 통합 토큰) 넣기.
- `notion_sync.page_text(page_id, token)` 로 페이지 평문 읽기, `extract_page_id(url)`로 링크→id.
- ⚠️ 그 노션 페이지가 **통합(MCP)에 연결**돼 있어야 읽힘(아니면 404). co-writer에 이미 연결돼 있으면 같은 토큰이라 그대로 됨.

## 명령/스레드에서 작품 잡는 패턴 (co-writer와 동일하게)
```python
# SUB_RE = re.compile(r"^\s*<\s*([^>]+?)\s*>\s*(.*)$", re.S)
work = None
wm = SUB_RE.match(q)
if wm:
    work = works.resolve(wm.group(1).strip()) or wm.group(1).strip()   # ← 별칭 해석 필수
    q = wm.group(2).strip()
if not work:                                  # <작품> 없으면 스레드에서 회수
    joined = "\n".join(m["content"] for m in _thread_messages(channel, thread_ts))
    w = _work_from_thread(joined)             # 노션 링크→작품 / <작품> 토큰(URL 제외)
    if w:
        work = works.resolve(w) or w
```
`_work_from_thread`는 [핸드오프_작품명끌어오기.md](핸드오프_작품명끌어오기.md) §3 참고(그대로 복사).

## 스토리보드 봇이 대본을 얻는 경로 (참고)
storyboard-bot은 보통 그 작품의 **대본**이 필요하다. 두 소스 중 택:
- **시트**(co-writer가 채운 빠른 캐시): `SheetBible.get(work)['scripts']['N화']` — 단, storyboard-bot도 같은 시트 웹앱 URL/secret 필요.
- **노션 직접**: `notion_sync.page_text(works.page_of(work))` 후 파싱.
작품명을 `works.resolve`로 정규화한 뒤 조회해야 별칭/정식명 둘 다 걸린다.
