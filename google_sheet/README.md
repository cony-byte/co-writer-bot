# 구글 시트 바이블 설정

입력은 슬랙 봇, 열람은 구글 시트. 봇이 [작품명] 지정 시 시트에서 바이블을 읽어 생성에 반영하고,
생성 결과를 시트에 되저장한다. 사람은 시트를 **보기만** 하면 된다.

## 세팅 (5분)

1. **구글 시트 새로 만들기.** 시트 탭 이름을 `bible`로 바꾼다. (A1~D1에 `work / kind / content / updated_at` 헤더를 넣어두면 보기 좋음 — 없어도 스크립트가 자동 생성)
2. **확장 프로그램 → Apps Script** 열기 → 기본 코드 지우고 [`Code.gs`](Code.gs) 전체 붙여넣기.
3. `Code.gs` 상단 `SECRET` 값을 **긴 무작위 문자열**로 바꾼다 (예: `openssl rand -hex 24` 결과). 이 값을 기억.
4. **배포 → 새 배포 → 유형: 웹 앱** → 실행: `나`, 액세스: `모든 사용자` → 배포. (권한 승인 요청이 뜨면 허용)
5. 나온 **웹 앱 URL**(`.../exec`로 끝남)을 복사.
6. 봇 `.env`에 두 값 입력:
   ```
   SHEET_WEBAPP_URL=https://script.google.com/macros/s/xxxx/exec
   SHEET_SECRET=3단계에서_넣은_그_문자열
   ```
7. 봇 재시작(또는 `새로고침`). 이제 `@co-writer [작품X] …`가 시트와 연동된다.

## 보안 메모

- `SECRET`은 URL을 아는 제3자의 무단 read/write를 막는 게이트다. 유출되면 `Code.gs`에서 값을 바꾸고 재배포 → `.env`도 갱신.
- 웹앱 액세스를 "모든 사용자"로 두는 이유: 봇 서버가 구글 로그인 없이 호출하기 위함. 실제 접근은 `SECRET`으로 통제된다.
- 시트에는 작품 설정·대본이 쌓이므로, 시트 공유 범위를 작가팀으로 제한할 것.

## 시트 구조

| work(작품명) | kind(구분) | content(내용) | updated_at |
|---|---|---|---|
| 작품X | 현재화 | 24 | … |
| 작품X | 로그라인 | … | … |
| 작품X | 인물 | … | … |
| 작품X | 24화_대본 | … | … |

- 업서트 키 = (work, kind). 같은 작품+구분이면 그 행을 최신본으로 덮어씀.
- kind: `현재화 · 로그라인 · 타겟정서 · 인물 · 줄거리 · 회차표 · N화_개요 · N화_대본 · 기획안`.
- 사람이 시트에서 직접 편집해도 다음 조회 때 봇이 읽는다(기본 입력구는 슬랙 봇).
