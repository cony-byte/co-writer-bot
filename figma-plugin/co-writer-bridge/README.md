# co-writer 스틸컷 브릿지 (피그마 플러그인)

영상화가 안전필터에 걸린 스틸컷을, 슬랙에서 `🎨 피그마로 보내기` 버튼을 누르면 피그마 캔버스로
가져와서 직접 손볼 수 있게 해주는 로컬 플러그인.

## 왜 이런 구조인가

피그마 REST API는 파일을 읽는 용도뿐이라(쓰기 엔드포인트 없음), 봇이 캔버스에 직접 이미지를
넣을 방법이 없다. 그래서 봇은 이미지를 로컬 대기열(`~/.co-writer-figma-queue/`, `SB_FIGMA_QUEUE_DIR`로
변경 가능)에 쌓아두고 작은 HTTP 서버(기본 `http://127.0.0.1:8933`)로 노출하기만 하고, 이
플러그인이 그 서버를 몇 초마다 폴링해서 실제로 캔버스에 이미지를 삽입한다.

**따라서 봇 프로세스와 피그마 데스크톱 앱이 이 포트(8933)에 서로 접근 가능한 위치(보통 같은
맥)에 있어야 동작한다.**

## 설치 (한 번만)

1. 봇 쪽 환경변수에 `SB_FIGMA_BRIDGE_ENABLED=true`를 설정하고 봇을 재시작한다(기본은 꺼져있음).
2. 피그마 데스크톱 앱에서: 메뉴 → Plugins → Development → **Import plugin from manifest…**
3. 이 폴더(`figma-plugin/co-writer-bridge/`)의 `manifest.json`을 선택한다.
4. 손보고 싶은 스틸컷 파일을 피그마에서 열어두고, Plugins → Development → **co-writer 스틸컷
   브릿지**를 실행한다 — 작은 창이 뜨고, 몇 초마다 자동으로 대기 중인 스틸컷을 확인한다.

## 사용

1. 슬랙에서 영상화가 안전필터로 실패하면 `🎨 피그마로 보내기` 버튼이 함께 온다.
2. 버튼을 누르면 봇이 그 컷의 원본 스틸컷을 대기열에 올린다.
3. 피그마에서 이 플러그인 창이 열려 있으면(백그라운드에서 계속 폴링 중이면) 몇 초 안에
   캔버스에 자동으로 이미지가 얹힌다 — 씬/컷 번호가 레이어 이름에 붙는다.
4. 캔버스에서 그 이미지를 직접 손본다(같은 노드를 편집해도 되고, 다른 이미지로 채워도 된다).
5. **손본 노드를 선택**하고 플러그인 창의 **↩️ 선택한 이미지 봇으로 보내기** 버튼을 누른다.
6. 봇이 그 컷의 원본 스틸컷 파일을 손본 이미지로 자동 덮어쓰고, 슬랙 스레드에 "✅ 반영했어요"
   라고 알려준다 — 그 다음 평소처럼 "이 컷 영상 만들어줘"/재생성을 시키면 손본 이미지로
   다시 영상화를 시도한다(새 명령어 없음, 기존 재생성 흐름 그대로).

## 제약

- 플러그인 창이 열려 있어야 폴링/되돌리기가 된다(피그마 플러그인은 창을 닫으면 실행이 멈춘다)
  — 상시 자동 반영을 원하면 이 부분은 추후 피그마 위젯(Widget) API로 바꾸는 게 나을 수 있다.
- 봇과 피그마가 다른 머신이면 포트를 열거나 SSH 터널링이 필요하다.
- "봇으로 보내기"는 이 플러그인이 직접 삽입한 노드(레이어)에서만 동작한다 — 원본 레이어를
  삭제하고 완전히 새로 만든 노드는 인식 못 하니, 원래 노드를 편집하거나 원래 노드의 이미지
  채우기(fill)만 바꾸는 걸 권장.
