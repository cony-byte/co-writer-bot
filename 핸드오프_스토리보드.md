# 핸드오프 — [스토리보드] 기능 (2026-07-09)

대본 → 숏폼(9:16 · 화당 ~90초 · 다인물) **씬 단위 스토리보드**를 슬랙에서 만드는 기능.
`[변환]` 핸들러와 같은 골격 + 스레드 후속 = 수정 흐름(`_do_revise`) 재사용.

## 3단계 흐름 (같은 스레드, 후속 자연어로 수정 · 수정 시 "바뀐 씬만" 출력)

1. **씬 설계** — `[스토리보드] <작품>` (또는 대본 붙여넣기 / 스레드 직전 대본) → 씬 분할 + 시간만(상세 X).
   피드백 loop. 스레드에 `생성`(SB_GEN_RE) 치면 2단계로.
2. **작가 확인용 스토리보드** — 6칸: `제목 · 길이 · 장소 · 등장 · 화면(쉽게) · 대사`.
   - `화면(쉽게)` = 영상 용어(OTS·인서트·클로즈업 등) **금지**, 문외한이 읽고 장면을 떠올리는 줄.
   - **★최우선 = 대본 내용 불변**(사건·행동·순서·대사 안 바꿈, 대사 원문 보존). 실무자 최대 불만이 "대본 지문과 달라짐"이라 이게 1순위.
3. **씨댄스(Seedance) 프롬프트** — ★미구현, 우선순위 아님(미룸).
   목표 산출물 = `~/Downloads/_에피소드01/스토리보드.md` 형식(2단계 6칸 + 액션·카메라·무드/조명 + 제작필드: `<<<element_id>>>` 인물 태깅·장소 element·씬 스틸 gpt_image_2·9:16·확정영상/레퍼런스). 파이프라인 = 이미지(스틸)→영상.
   착수 전 확인: 씨댄스가 이미지→영상인지 텍스트→영상인지.

## 파일

- `bot/video_guide.py` — 가이드 SSOT(`VIDEO_GUIDE` 삼중따옴표 상수). 교체는 여기만. **본문에 삼중따옴표 금지**(백틱은 안전).
  - 0.절대원칙(대본 불변) → 1.분량 → 2.작가확인용 6칸 → 3.좋은예(EP01) → 4.셀프체크 → **[부록] 씨댄스(3단계)용 카메라·무드 문법(2단계엔 안 씀)**.
- `bot/prompts.py` — `storyboard_plan_system`(1단계) / `storyboard_system`(2단계) / `storyboard_bible_block`(작품 바이블 주입) / `STORYBOARD_FEEDBACK_PRINCIPLE`(문외한 자연어 해석). 가이드 비면 `RuntimeError`.
- `app.py` — `CMD_STORYBOARD`, `_do_storyboard`(1단계), `_do_revise`의 `sb` 모드(생성/설계수정/상세수정 분기), `_sb_stage`(plan/detail 감지: 마커 `화면(쉽게)` vs `씬 설계안`), `SB_GEN_RE`.

## 특징

- **문외한 피드백 해석**: "씬4 임팩트 좀" 같은 막연한 요청 → 의도 해석해 구체 반영 + `바꾼 점:` 한 줄로 쉽게 설명.
- **작품 바이블(노션→시트 동기화분) 물림**: `<작품>` 주면 금지사항·현재화/해당화 개요·로그라인·핵심정서·타겟·회차분배·등장인물 주입(`storyboard_bible_block`). `<작품>` 없으면 대본만으로.
