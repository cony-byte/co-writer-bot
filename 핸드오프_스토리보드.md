대본 → 숏폼(9:16 · 화당 ~90초 · 다인물) 스토리보드를 슬랙에서 만드는 기능.
`[변환]` 핸들러와 같은 골격 + 스레드 후속 = 수정 흐름(`_do_revise`) 재사용.

## 2단계 흐름 — 단계를 '명령'으로 지정, 명령이 스레드를 직접 읽음 (끊김 방지)

> ⚠️ 예전 '작가 확인용 화면쉽게(6칸)' 중간단계는 **폐기**(실무자가 GPT 이미지 결과로 판단하지 텍스트로 확인 안 함).
> ⚠️ 예전 방식(스레드에 바레 `생성` → 봇이 단계 눈치껏 판별)은 자꾸 끊겨서 **명시 명령**으로 바꿈. `_do_revise`의 바레-후속 경로는 보조로 남아있음.

1. **`[스토리보드1] <작품> N화`** = 씬 설계 (분할·시간). 대본 확보: 시트(노션) 그 화 대본 → rest 붙여넣기 → 스레드 직전 봇 출력.
   수정: `[스토리보드1] <작품> 씬3 8초로` — 스레드의 설계안을 읽어 **바뀐 씬만** 재출력.
2. **`[스토리보드2] <작품>`** = 상세 콘티 (GPT 이미지용). 스레드의 **확정 씬 설계안 + 원본 대본(시트)** 을 읽어, 대본을 **샷 단위 연속 산문**으로. 각 샷: 프레임에 잡히는 것(괄호)·카메라 무빙·대사 원문·나레이션 처리(입 안 움직임)·연기 뉘앙스.
   수정: `[스토리보드2] <작품> 씬3 더 세게` — 스레드의 콘티를 읽어 바뀐 샷/구간만.
   - **★최우선 = 대본 내용 불변**(사건·행동·순서·대사 안 바꿈, 대사 원문). 실무자 최대 불만이 "대본과 달라짐".
   - 정본 예시 = `video_guide.py §3`(실무자가 GPT 이미지로 가장 만족한 EP01 연우방 콘티).
   - (그 다음 = GPT 이미지 → 씨댄스 영상. 봇 밖/추후.)

## 파일

- `bot/video_guide.py` — 가이드 SSOT(`VIDEO_GUIDE` 삼중따옴표 상수, v2). 교체는 여기만. **본문에 삼중따옴표 금지**(백틱·일반따옴표 안전).
  - 0.대본불변 → 1.상세콘티=샷단위연속산문(명시사항 5개) → 2.프레이밍·카메라 문법 → 3.정본예시(EP01 연우방) → 4.셀프체크.
- `bot/prompts.py` — `storyboard_plan_system`(1단계 씬설계) / `storyboard_system`(2단계 **상세 콘티**) / `storyboard_bible_block`(작품 바이블 주입, 개요는 안 넣음) / `STORYBOARD_FEEDBACK_PRINCIPLE`(문외한 자연어 해석). 가이드 비면 `RuntimeError`.
- `app.py` — `CMD_STORYBOARD`(1단계)·`CMD_STORYBOARD2`(2단계), `_do_storyboard(…, stage)`(단계별 · 스레드 직접 읽음 · 시트 대본 자동 `_sb_script_from_bible` · 이전 설계안/콘티 회수 `_last_assistant_with`), 배지 `SB_BADGE_PLAN/BOARD`([1단계]/[2단계]). `_do_revise`의 `sb` 바레-후속 경로·`_sb_stage`·`SB_GEN_RE`는 보조로 잔존.

## 특징
